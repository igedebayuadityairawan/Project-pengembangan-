1 Latar Belakang & Konsep Project

Sistem absensi konvensional yang masih menggunakan tanda tangan manual pada kertas memiliki banyak kelemahan, antara lain: potensi kecurangan (titip absen), proses rekap data yang memakan waktu, serta sulitnya mendapatkan data secara real-time. Sistem ini dirancang sebagai solusi modern menggunakan pendekatan Internet of Things (IoT) dan konsep Wireless Sensor Network (WSN).

Project ini merupakan Sistem Absensi Otomatis Berbasis IoT yang menggunakan mikrokontroler ESP32 sebagai Sensor Node, modul RFID RC522 sebagai pembaca kartu identitas, dan MySQL sebagai database penyimpanan data absensi. Sistem mampu mengirim data secara wireless melalui WiFi ke server lokal (XAMPP) dan menampilkan hasilnya pada Web Dashboard secara real-time.

💡 Konsep Inti: Setiap pengguna memiliki kartu RFID dengan UID unik. Saat kartu di-tap pada reader, ESP32 membaca UID tersebut dan mengirimnya ke server PHP via HTTP POST. Server mencatat absensi ke database MySQL beserta informasi sesi (Pagi/Siang/Sore/Malam) secara otomatis berdasarkan waktu.

2 Tujuan Sistem

  1. Membangun sistem absensi yang otomatis, efisien, dan tidak dapat dipalsukan menggunakan RFID.
  2. Menerapkan konsep Wireless Sensor Network (WSN) dimana ESP32 berperan sebagai Sensor Node dan server sebagai Sink Node / Gateway.
  3. Menyediakan Web Dashboard berbasis PHP untuk monitoring absensi secara real-time dengan fitur notifikasi, statistik, dan manajemen data pengguna.
  4. Memberikan feedback interaktif kepada pengguna melalui LCD I2C (visual) dan Buzzer (audio) langsung pada perangkat.
  5. Menerapkan klasifikasi sesi absensi otomatis berdasarkan waktu (Pagi, Siang, Sore, Malam) menggunakan sinkronisasi NTP.

3 Arsitektur IoT (3 Layer)

Sistem ini menggunakan arsitektur IoT standar 3 layer yang umum digunakan pada implementasi Wireless Sensor Network:

🔌 Layer 1 — Perception Layer (Sensor Node)

Layer paling bawah yang bertugas mengumpulkan data dari dunia fisik. Pada sistem ini terdiri dari:

  • ESP32 DevKit V1 — Mikrokontroler utama (Sensor Node) dengan WiFi built-in
  • MFRC522 RFID Reader — Membaca UID kartu RFID 13.56 MHz via protokol SPI
  • LCD I2C 16x2 — Menampilkan informasi status, waktu, dan feedback ke pengguna via protokol I2C
  • Active Buzzer — Feedback audio (beep sukses/gagal) via PWM pada GPIO4
📡 Layer 2 — Network Layer (Komunikasi)

Layer tengah yang mengatur transmisi data dari Sensor Node ke Server:

  • WiFi 802.11 b/g/n — Koneksi wireless ESP32 ke jaringan lokal (Access Point)
  • HTTP POST — Protokol pengiriman data UID dari ESP32 ke PHP API endpoint
  • NTP (Network Time Protocol) — Sinkronisasi waktu ESP32 dengan server pool.ntp.org
  • JSON / Plain Text — Format respon data antara server dan client (web dashboard)
🖥️ Layer 3 — Application Layer (Server & Dashboard)

Layer teratas yang memproses, menyimpan, dan menampilkan data:

  • Apache Web Server (XAMPP) — Menjalankan backend PHP dan menyajikan halaman web
  • PHP Backend API — Memproses data UID, menentukan sesi, menyimpan ke database (berperan sebagai Sink Node / Gateway)
  • MySQL Database — Menyimpan data pengguna RFID dan log absensi (berperan sebagai Cloud Database)
  • Web Dashboard — Interface monitoring real-time dengan statistik, notifikasi, dan manajemen user

SCK GPIO 18 Serial Clock
MOSI GPIO 23 Master Out Slave In
MISO GPIO 19 Master In Slave Out

6.2 Koneksi LCD I2C 16x2 → ESP32 (I2C)

Pin LCD I2C Pin ESP32 Keterangan
SDA GPIO 21 I2C Data (default ESP32)
SCL GPIO 22 I2C Clock (default ESP32)
VCC VIN (5V) Power supply via USB 5V
GND GND Ground

6.3 Koneksi Buzzer → ESP32

Pin Buzzer Pin ESP32 Keterangan
Positif (+) GPIO 4 PWM output (2700 Hz, 8-bit resolution)
Negatif (–) GND Ground
⚠️ PENTING: Modul RFID RC522 hanya boleh diberi tegangan 3.3V. Menghubungkan ke 5V dapat merusak modul secara permanen! Pin RST RFID dipindah ke GPIO 27 karena GPIO 22 sudah digunakan untuk I2C SCL.

8 Alur Kerja Sistem (System Flow)

8.1 Proses Booting ESP32

  1. ESP32 dinyalakan → Inisialisasi LCD, SPI, RFID Reader
  2. LCD menampilkan "Connecting WiFi" → ESP32 terhubung ke Access Point
  3. LCD menampilkan IP Address yang didapat → Buzzer beep sukses
  4. Sinkronisasi waktu NTP (pool.ntp.org, UTC+8 WITA)
  5. LCD menampilkan "RFID ABSENSI" + jam + sesi → Siap digunakan

8.2 Proses Tap Kartu RFID

  1. Pengguna mendekatkan kartu RFID ke reader RC522
  2. ESP32 membaca UID kartu (4 byte → 8 hex) via SPI
  3. Buzzer beep pendek sebagai konfirmasi baca
  4. LCD menampilkan "UID: XXXXXXXX — Mengirim data..."
  5. ESP32 mengirim HTTP POST ke getUID.php dengan parameter:
    • apikey = kunci API untuk validasi
    • UIDresult = UID kartu yang dibaca
  6. Server PHP memproses:
    • Validasi API Key
    • Menentukan sesi berdasarkan jam server
    • Menyimpan log absensi ke database MySQL
    • Mencari nama pengguna berdasarkan UID
    • Mengirim respon: OK|NamaMahasiswa|Sesi
  7. ESP32 menerima respon → Parse data nama & sesi
  8. LCD menampilkan greeting + nama pengguna (2 detik)
  9. LCD menampilkan info absensi sesi + UID (2 detik)
  10. Kembali ke layar utama (idle)
✅ Respon Server: Format OK|NamaMahasiswa|Sesi — contoh: OK|I Gede Surya|Pagi

8.3 Kondisi Error

Kondisi Respon ESP32 LCD Display Buzzer
UID belum terdaftar Response contain "UNKNOWN" ❌ Blm Terdaftar 3× beep pendek
Server error / timeout HTTP code ≠ 200 ❌ TAP GAGAL! Server Error 3× beep pendek
WiFi putus WL_DISCONNECTED ❌ WiFi PUTUS! 1× beep panjang (700ms)

10 Logika Sesi Absensi

Sistem menentukan sesi absensi secara otomatis berdasarkan jam saat tap kartu. Logika sesi berjalan baik di ESP32 (via NTP) maupun di server PHP:

Sesi Rentang Waktu Greeting LCD Badge Warna
🌅 Pagi 06:00 – 11:59 "Selamat Pagi!" ● Kuning
☀️ Siang 12:00 – 14:59 "Selamat Siang!" ● Biru
🌇 Sore 15:00 – 17:59 "Selamat Sore!" ● Orange
🌙 Malam 18:00 – 05:59 "Selamat Malam!" ● Ungu

Kode Logika Sesi (ESP32 & PHP)

// ESP32 (C++) — Sinkronisasi NTP UTC+8
String getSesi(int hour) {
  if (hour >= 6  && hour < 12) return "Pagi";
  if (hour >= 12 && hour < 15) return "Siang";
  if (hour >= 15 && hour < 18) return "Sore";
  return "Malam";
}

// PHP — Timezone Asia/Makassar (WITA UTC+8)
$hour = (int) date('G');
if ($hour >= 6  && $hour < 12) $sesi = 'Pagi';
elseif ($hour >= 12 && $hour < 15) $sesi = 'Siang';
elseif ($hour >= 15 && $hour < 18) $sesi = 'Sore';
else $sesi = 'Malam';

12 Kesimpulan

Sistem Absensi Otomatis Berbasis IoT ini berhasil mengintegrasikan beberapa komponen utama dalam konsep Wireless Sensor Network (WSN):

  • ESP32 sebagai Sensor Node yang membaca data RFID dan mengirimkannya secara wireless
  • PHP Backend sebagai Sink Node / Gateway yang menerima dan memproses data dari sensor node
  • MySQL Database sebagai Cloud Database untuk penyimpanan data absensi
  • Web Dashboard sebagai Application Layer untuk monitoring dan manajemen sistem

Dengan arsitektur 3 layer IoT (Perception, Network, Application), sistem ini menyediakan solusi absensi yang otomatis, real-time, efisien, dan user-friendly dengan feedback visual (LCD) dan audio (Buzzer) yang interaktif.

✅ Keunggulan Sistem:
  • Tidak bisa dipalsukan — UID RFID unik dan sulit diduplikasi
  • Real-time monitoring — Dashboard update otomatis setiap beberapa detik
  • Sesi otomatis — Klasifikasi waktu absensi tanpa input manual
  • Feedback interaktif — LCD + Buzzer memberikan konfirmasi langsung
  • Low-cost — Menggunakan komponen yang mudah didapat dan terjangkau
